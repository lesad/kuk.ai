## CLI Surface

```
peep <design> <impl> [--output <path>] [--threshold <f64>] [--fail] [--json] [--no-diff]
```

Defaults: `--output diff.png`, `--threshold 0.01` (hybrid score scale: 0 = identical, 1 = max diff), no `--fail`.

Human stdout:
```
score: 0.0042 (99.58% similar)
diff:  diff.png
```

JSON stdout:
```json
{"score":0.0042,"similarity":0.9958,"threshold":0.01,"passed":true,"diff_path":"diff.png"}
```

Errors â†’ stderr, exit 2 (distinct from `--fail`'s exit 1).

## Architecture

```
src/
  main.rs       clap parse, orchestrate, set exit code
  cli.rs        Args struct (derive)
  compare.rs    load PNG/JPG, validate dims, call rgba_hybrid_compare â†’ CompareResult
  overlay.rs    build red overlay from SimilarityImage + impl pixels â†’ save PNG
  report.rs     Human + JSON renderers from CompareResult
  error.rs      thiserror enum (ImageLoad, DimMismatch, Encode, Io)
```

One responsibility per module. `main.rs` stays thin (parse â†’ compare â†’ overlay â†’ report â†’ exit).

### Data flow

1. `cli::Args` parsed
2. `compare::run(design_path, impl_path)` â†’ loads both via `image::open`, verifies `dimensions()` equal else `Err(DimMismatch)`, calls `image_compare::rgba_hybrid_compare(&design_rgba, &impl_rgba)?`, returns `CompareResult { score, similarity_image, width, height }`
3. If diff output enabled: `overlay::render(&impl_img, &similarity_image, RED)` â†’ `RgbaImage`, save to `--output`
4. `report::emit(&result, &args)` â†’ stdout (human or JSON)
5. exit: `2` on error, `1` if `--fail && score > threshold`, else `0`

### Overlay rendering

`SimilarityImage` from image-compare is a single-channel diff map (per-pixel, 0..1 scale). For each pixel: `alpha = clamp(diff * gain, 0, 1)` where `gain` boosts visibility (start with 4.0, tune empirically). Blend `(255, 0, 0)` over impl pixel using `alpha`. Output `RgbaImage`, write PNG via `image` crate.
